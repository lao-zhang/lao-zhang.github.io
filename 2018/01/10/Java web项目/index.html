传统java web项目

经过系统的理论学习和实践，鄙人对于不依赖于框架的传统java web工程有了一定的认识与掌握，现在结合具体项目来对传统java web工程整个构建工程进行分析。

一、概述

在我看来，一个传统的java web项目由数据库、逻辑事务处理、视图页面三部分组成。

数据库：

​ 用来存放一定量的数据。在我们写程序时，如果只是实现一个算法式的程序，所需要的数据通常经由键盘输入，这些数据具有不复用、数量较小、类型单一的特性。但如果实现处理现实事务的复杂程序时（比如记账本或者学生信息管理系统），涉及到的数据不仅数量较多，类型丰富，而且需要反复使用，这就需要程序跟数据库连接来读写数据了。

​ 数据库方面的操作包括：环境配置、创建数据库、创建表等。

逻辑事务处理：

​ 这是整个工程的核心所在，对内要构建对象、设计方法、处理数据，对外要连接数据库、跟前端进行数据通信。

​ 逻辑事务处理方面的操作包括：java代码的编写、xml文件的编写、部分jsp和html页面的编写等。

视图页面：

​ 跟用户最为密切一部分，用户通过前端（或称客户端）输入数据或者查看数据，通常由jsp或者html构成。

​ 视图页面的工作有：向逻辑事务处理环节发送表单、读取逻辑事务处理环节发送来的信息、一些javascript的编写、界面美观等。

###二、项目概述

​ 简单而具有代表性的项目便于说明问题，所以这里以一个大家很熟悉的个人记账项目作为说明的对象。

​ 个人账单的功能有：添加消费记录、查看所有消费记录、设置当月预算、消费分类、消费统计

​ 体现到具体程序中：数据库数据添加、修改、查询（未涉及到删除），使用java语言对数据进行一些简单处理等。

​ 总体说来，这个项目十分简单，但是具备了一个java web工程的必须成分，麻雀虽小，五脏俱全。
